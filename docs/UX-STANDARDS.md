# UX 设计规范

规则级别：**强制**必须遵守，**建议**按场景采用。现有模块的稳定信息架构优先；调整布局前先读相邻页面和模块设计。

## 1. 布局与滚动

- **强制**：页面根容器使用稳定的 `h-full/min-h-0/overflow-hidden` 约束，只让内容区或列表滚动。
- **强制**：主内容使用 `min-w-0 flex-1`，列表使用 `min-h-0 flex-1 overflow-y-auto`；工具栏 `shrink-0`。
- **强制**：多子功能页面使用既有的侧栏或顶部 Tab 模式，不在同一模块混用；内容优先于装饰性 Card。
- **建议**：标题和主要操作保持单行紧凑工具栏；重复项才使用 Card，页面 section 不套 Card。
- **建议**：固定格式控件和列表项使用稳定尺寸、grid track 或 aspect ratio，避免动态内容导致布局跳动。

## 2. 加载与进度

- **强制**：首次异步加载显示与最终布局匹配的 skeleton，不留空白或只显示孤立 spinner。
- **强制**：只有后端提供可信 `completed/total` 时显示百分比；未知总量用 indeterminate，禁止伪造进度。
- **强制**：刷新保留旧数据并显示紧凑刷新状态，不用全屏 overlay 覆盖已有内容。
- **强制**：complete、partial、unsupported、failed、cancelled 必须可区分；失败不得折叠为空结果成功。
- **强制**：可取消任务提供 Cancel/Cancelling 状态；进入不可取消临界区时明确反馈。
- **强制**：冲突写操作在刷新/任务期间禁用；只读浏览是否可继续由任务一致性策略决定。

## 3. 异步与性能

- **强制**：Rust 阻塞 I/O 进入 `spawn_blocking` 或等价阻塞线程池；并发必须有上限、取消和错误汇总。
- **强制**：可重复点击的异步动作做程序化重入保护，不能只依赖按钮 disabled。
- **强制**：长列表评估虚拟化；图标、图片和昂贵详情只按可见项加载。
- **建议**：传给重组件的派生数据、配置和回调保持引用稳定，但不为低成本渲染机械添加 memo。

## 4. 文本与国际化

- **强制**：用户文案来自 i18n；控件在中文、英文和系统字体下均不得溢出或遮挡。
- **强制**：单行可省略文本使用 `min-w-0 truncate` 并提供 `title`/Tooltip；需要理解全文时允许换行，不能一律截断。
- **强制**：长命令使用图标按钮与 Tooltip；常见操作优先 Lucide 图标，不手绘 SVG。
- **强制**：按钮、标签和计数器有稳定尺寸；最长单词仍无法容纳时调整布局，不按 viewport 缩放字号。

## 5. 状态与操作

- **强制**：列表和详情必须有 loading、empty、error 和 retry 状态。
- **强制**：写操作展示进行中、成功和失败反馈；输入被过滤或拒绝时说明原因。
- **强制**：删除、清理、kill、重置权限等危险操作使用 `DestructiveConfirmDialog`，说明不可逆后果和范围。
- **强制**：icon-only 按钮有可访问名称和 Tooltip；自定义 checkbox/开关需键盘可操作并提供正确 ARIA。
- **建议**：辅助统计用紧凑内联布局；操作按钮使用 `xs/sm`，避免大面积营销式标题和卡片。

## 6. 验收

至少检查：首次加载、刷新保留旧数据、空/失败/partial、快速连点、取消、长文本、中英文切换、键盘操作、窗口缩放和大数据量。涉及 Canvas/3D/复杂响应式界面时用真实截图验证无空白和重叠。
