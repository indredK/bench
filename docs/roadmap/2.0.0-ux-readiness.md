# Bench 2.0.0 用户体验完善与验收规范

> 目标版本：2.0.0  
> 文档状态：发布门禁  
> 功能与发布规范：[2.0.0 功能完善与发布就绪](./2.0.0-release-readiness.md)  
> 通用规范：[UX-STANDARDS.md](../UX-STANDARDS.md)

本文负责把已经存在的功能变成稳定、可理解、可恢复的桌面体验。它不要求增加营销页、教程页或新功能类别；重点是核心任务的效率、真实异步反馈、响应式、国际化、可访问性、性能和视觉一致性。

## 1. AI 执行规则

1. 先完成对应功能工作包的 contract，再做 UI。后端没有可信 total 时不能伪造百分比；后端不区分 partial 时不能只靠前端文案补救。
2. 修改页面前读取 `docs/UX-STANDARDS.md`、相邻页面、现有 shadcn/ui 组件和 `src/styles/theme/tokens.css`。
3. 优先扩展 `ContentView`、`FeatureLoadError`、`RuntimeFeatureGate`、`DestructiveConfirmDialog`、`ThreeColumnLayout` 等现有模式。只有出现三个以上真实使用方时才新增共享抽象。
4. 不把页面 section 包成装饰性浮动 Card，不嵌套 Card；工作台保持安静、紧凑、可扫描。圆角不超过既有设计系统默认值。
5. 命令按钮优先 Lucide 图标；icon-only 必须有 accessible name 和 Tooltip。不要用带文字的圆角矩形替代用户熟悉的刷新、返回、关闭、复制等图标。
6. 不在界面放“本页面支持什么”“如何使用”“快捷键说明”等功能介绍文字。必要提示只解释当前状态、风险或下一步。
7. 每次只完成一个 Uxx 工作包；用真实 fixture 渲染 loading/empty/partial/error，而不是只截图 happy path。
8. UI 完成必须同时有自动化、截图/像素检查和键盘操作证据。看起来正常但文本溢出、焦点丢失或失败态不可恢复，仍视为未完成。

## 2. 2.0.0 UX 目标

Bench 面向高频使用开发工具的用户。2.0.0 体验应满足：

- **快**：启动后立即看到稳定壳层；搜索、筛选、切换不阻塞；大列表虚拟化。
- **真**：显示真实 loading、stale、partial、unsupported、failed、cancelled，不用成功 toast 掩盖后台失败。
- **稳**：加载和动态内容不改变工具栏、列表和详情的基本几何；窗口缩放没有内容重叠。
- **省操作**：核心路径可用键盘完成，常用操作靠近对象，刷新保留上下文和旧数据。
- **可恢复**：区域错误有 retry；取消后保留可信数据；危险操作说明影响范围。
- **跨语言/平台一致**：中文、英文、macOS、Windows 只在能力和系统约定上不同，不在状态语义上不同。

## 3. 统一异步状态语言

所有数据页面必须能表达下列状态。不得把多个状态折叠成一个布尔 `loading` 后靠空数组猜测。

| 状态 | 页面行为 | 禁止行为 |
|------|----------|----------|
| initialLoading | 与最终布局同尺寸 skeleton，页面根 `aria-busy=true` | 空白页、孤立 spinner、闪烁营销式 loading |
| refreshing | 保留旧数据，工具栏/区域显示紧凑 indeterminate 或真实进度 | 全屏遮罩旧数据、清空选择和滚动位置 |
| ready | 展示最新可信数据和必要的更新时间 | 每次成功都弹 toast 干扰高频操作 |
| stale | 继续显示旧数据，同时标注更新时间和刷新入口 | 伪装为最新、自动删除旧数据 |
| empty | 说明当前数据域为空，并给出唯一合理主操作 | 只有“无数据”而没有上下文 |
| filterEmpty | 保留筛选器，说明无匹配并提供清除筛选 | 误导用户去新增数据 |
| partial | 数据正常可浏览，区域 warning 汇总失败数并可查看/重试 | toast 后消失、把失败项从列表删除 |
| failed | 区域内错误、原因和 retry；首载失败使用 `FeatureLoadError` | 空数组、只写 console、错误覆盖整个应用壳 |
| unsupported | `RuntimeFeatureGate/DesktopOnly` 说明平台不支持 | 显示为空、显示开关为 false |
| cancelling | 保留内容，取消按钮禁用并显示进行中 | 允许重复取消、立刻假装已取消 |
| cancelled | 返回稳定旧状态，显示可再次启动 | 合并进 failed、丢失已完成结果 |

建议在 controller/use-case 建立领域状态，不强制创建全局 mega store：

```ts
type AsyncDataState<T> =
  | { kind: "initialLoading" }
  | { kind: "ready"; data: T; refreshedAt: number }
  | { kind: "refreshing"; data: T; progress?: { completed: number; total: number } }
  | { kind: "partial"; data: T; failures: FailureSummary[]; refreshedAt: number }
  | { kind: "stale"; data: T; reason: string; refreshedAt: number }
  | { kind: "failed"; error: LocalizedError }
  | { kind: "unsupported"; reasonCode: string }
```

只有后端提供可信 `completed/total` 时显示百分比。未知总量使用 indeterminate；不能根据定时器平滑到 90%。

## 5. U00：应用壳、启动和导航

修改入口：

- `src/App.tsx`、`src/main.tsx`、`src/splash.ts`
- `src/components/layout/{NavigationShell,Sidebar,TopTabNavigation,BottomTabNavigation,CustomTitlebar}.tsx`
- `src/hooks/useNavigationLayout.ts`
- `src/features/registry.tsx`

要求：

1. Splash 只在主窗口未 ready 时存在；ready event 先后竞态、listener 失败和 fallback timer 都有测试。关闭 splash 到首个页面之间不得出现可见透明空窗。
2. 壳层先于慢数据可交互；启动问题以紧凑 alert 显示，不阻塞不相关模块。
3. 导航只显示当前平台适用 feature；旧 route/书签命中 unsupported 时有返回可用页面的操作。
4. 侧栏、顶部 Tab、底部导航使用同一 active/disabled/focus 语义；切换布局后当前 route 和焦点不丢失。
5. 默认窗口 1280×800，最小 960×600。所有布局在这两个尺寸和 1440×900、1920×1080 下无重叠；不通过 viewport 缩放字号。
6. 自定义标题栏在 macOS/Windows 分别验证 drag、double click、minimize/maximize/close、always-on-top；交互失败对用户可恢复，不能只 `console.error`。
7. 页面切换动画不超过现有短时长；`prefers-reduced-motion` 时取消位移/扰动。Scramble 等装饰不能阻止读取标题。
8. 路由 Suspense fallback 保持内容区几何，不让侧栏和标题栏跳动。

自动化：route 重定向、平台过滤、布局切换、startup issue、ready race、reduced motion。手工：三种导航布局在 macOS/Windows 用鼠标和键盘走一遍。

## 6. U01：通用状态与反馈组件

优先复用：

- `src/components/content/ContentView.tsx`
- `src/components/common/{FeatureLoadError,RuntimeFeatureGate,DestructiveConfirmDialog,StartupIssuesAlert}.tsx`
- `src/components/layout/{DetailPanel,FilterPanel,ThreeColumnLayout}.tsx`
- `src/components/ui/` 下 shadcn/ui 基元

要求：

1. `ContentView` 首载 skeleton 不再使用“35% 宽度动画”暗示虚假进度；indeterminate 使用不表达百分比的动画，真实进度才绑定 width。
2. 增加或统一 filterEmpty、partial、stale、cancelled 表达；不要把所有页面塞进一个过度通用组件，允许 feature 组合共享基元。
3. Error 必须在所属区域长期可见，toast 仅作补充。错误区包含本地化标题、可理解原因、retry 和必要的详情展开。
4. Empty state 文案说明数据域；主操作最多一个，次操作使用低强调样式。没有合理操作时不放按钮。
5. 写操作有程序化重入保护；按钮 disabled 只负责视觉反馈。成功反馈按操作影响选择 inline 更新或 toast，不两者重复。
6. 删除、清理、kill、重置权限等使用 `DestructiveConfirmDialog` 或符合相同 contract 的专用 Sheet：对象、数量、不可逆后果、确认动作清楚。
7. Toolbar、计数、筛选和进度使用固定 track/高度；loading 文案、长数字、状态 badge 不改变相邻布局。
8. Copy/reveal 等瞬时状态在切换对象/卸载时清理 timer；复制成功反馈不改变按钮宽度。

测试至少覆盖 initialLoading -> ready、initialLoading -> failed -> retry、ready -> refreshing -> partial、cancel -> stable、快速连点。

修改入口：`src/features/quick-launch/`、`src/shared/app-inventory/`、`src/components/content/VirtualGridView.tsx`。

## 8. U03：App Manager

核心用户路径：扫描 -> 筛选/选择 -> 查看详情 -> 启动/更新/安装/卸载 -> 查看结果。

修改入口：`src/features/app-manager/`、`src/components/content/ContentView.tsx`。

要求：

- 首次 inventory、刷新、provider partial、unsupported 和 failed 分区表达；warning 不得占满主内容并挤走应用列表。
- 列表/网格切换保留筛选、排序、选择和滚动上下文；长列表始终虚拟化。
- 详情区域显示来源 evidence、版本、路径/包 ID 的可理解摘要；高风险技术细节放可复制详情，不让用户判断模糊匹配是否安全。
- Update available、up-to-date、unsupported、provider failed 使用不同 badge/文案；未知不显示为已是最新。
- 批量操作有固定底部/顶部进度区，展示 total/succeeded/failed/cancelled；失败项可筛选和重试，成功项即时 merge。
- 安装/更新下载有真实进度或 indeterminate、速度/剩余时间仅在数据可信时显示；取消后恢复可操作状态。
- 卸载/覆盖/替换必须二次确认并说明具体应用和来源；无法建立 exact evidence 时动作不可用。
- 操作进行中只禁用冲突动作，详情浏览和无冲突筛选继续可用。

自动化必须覆盖 warning + list 同屏、batch partial、取消、目标 revision 过期、视图切换状态保持和长名称。

## 9. U04：Account Manager

核心用户路径：选择站点 -> 选择账号 -> 登录/恢复/刷新 -> 查看状态；外部授权和敏感信息必须让用户清楚发生了什么。

修改入口：

- `src/features/account-manager/page.tsx`
- `hooks/useAccountManagerController.ts`、`hooks/useAuthProxy.ts`
- `components/{StationColumn,AccountColumn,DetailColumn,dialogs,auth-proxy-dialog,external-apps-panel}.tsx`

要求：

1. 首载使用三栏 skeleton，不再只显示 `common.loading` 文本。站点、账号、详情 skeleton 与最终宽度一致。
2. 1280+ 使用三栏；960-1279 选中账号后用 Sheet/Drawer 展示详情，包含清晰返回、焦点恢复和 Esc 关闭。不能简单隐藏详情。
3. Station/Account 为空、筛选无结果、加载失败、刷新 partial、Session expired、LoginRequired、FetchFailed 分别表达。
4. 刷新一个/站点/全部显示作用范围；同账号 single-flight 时 follower 复用同一状态，不出现两个 spinner 或窗口。
5. 500+ 账号和大量站点使用虚拟列表；拖拽排序与虚拟化组合必须键盘可用，操作期间不改变行高。
6. 密码默认只显示“已设置”；主动 reveal 后倒计时清除。隐藏、切换账号、关闭弹窗立即清值；复制有短反馈但不显示密码。
7. Auth Proxy 对话框显示请求来源、目标 origin、callback app 和候选账号；不能展示完整含 token/code 的 URL。授权 ticket 过期/使用后关闭敏感操作。
8. 登录窗口“已打开”不是“登录成功”。UI 只在 completed + verified 后显示 Ready；pending、callback、capture、probe、failed 状态可区分。
9. 删除结果 partial 时列出未清理资源和 retry，不显示完整敏感路径。导入先预览版本、站点数、账号数和模式，再确认写入。
10. External Apps 显示授权范围、最近使用和撤销后果；撤销/关闭代理使用危险确认并报告结果。

必须测试：中英文、长站点/用户名、CJK/emoji、0/500+ 账号、快速连点刷新、ticket 过期、密码 TTL、窄屏 Sheet、删除 partial、Windows unsupported 子能力。

## 10. U05：Clean Space

核心用户路径：查看空间 -> 扫描 -> 选择可清理项 -> 确认 -> 清理 -> 查看结果。

修改入口：`src/features/clean-space/` 与复用的 `src/features/dev-cleaner/` 组件。

要求：

- 首载 skeleton 匹配磁盘摘要、分类列表和环形图；快照先到时标记 stale/estimated，精扫更新不重建整个页面。
- 扫描进度 total 不可信时 indeterminate；允许浏览旧数据，冲突清理动作禁用。
- 分类大小变化平滑但不伪造精确值；单位统一用 locale-aware formatter，避免 `MB/GB` 文本抖动。
- 可清理、受保护、未知三种语义明确。风险颜色不是唯一信号，必须有 icon/label。
- 选择栏稳定显示数量和预计空间；受保护项不可被“全选”偷偷包含。
- 确认 Sheet 列出范围、风险和不可逆后果；高风险项要求更强确认，但不使用输入魔法词增加无意义摩擦。
- 清理进度逐项更新；partial 后成功释放量只计成功项，失败项保留原因和重试。关闭进度页后记录仍可查。
- 自定义目录选择被拒绝时说明保护原因；路径显示按 Home 相对化并可复制，默认不暴露完整用户名。
- Windows 直达显示平台 unsupported，不启动 macOS 命令后再报错。

视觉上内容区使用未加框的分区和真实重复项 Card；不要把总览 section、分类 Card、内部工具再层层套 Card。

## 11. U06：开发工具、设置与本地模块

### Dev Toolbox

- Tab 切换保持每个工具输入/结果，首载只加载当前 Tab；低频大模块继续 lazy。
- 每个工具错误留在工具区域，不让一个 Tab 错误覆盖其他 Tab。
- JSON/正则/编码等编辑器使用等宽字体、清晰输入输出分区、可复制反馈和大文本上限；长内容不撑破页面。
- Browser preview 对无 IPC 的纯工具可用，桌面专属 Tab 显示 gate。

### Port Manager

- 扫描保留旧列表；端口变化以稳定 key 更新，不整表闪烁。
- Kill/释放使用危险确认，显示 PID、进程名、端口和影响；PID 已退出视为幂等完成，不显示灾难性错误。
- 大 PID 树虚拟化；权限失败、进程已变化、平台 unsupported 分开。
- 自动刷新期间用户筛选和详情可用，旧事件不覆盖新任务。

### Env Detector

- 初始提示、扫描中、partial、failed、empty 分开；每个 provider 可独立失败。
- 导出前显示会脱敏的字段类型；成功后给路径/打开位置操作，不泄露内容到 toast。
- 复制/导出按钮在无可信结果时禁用并说明原因。

### System Settings

- 读取中使用局部 skeleton/disabled，不把未知值显示为 off。
- `unknown/error/unsupported` 使用状态 icon 和说明；Switch 只用于已知布尔值。
- 写入时只让当前 setting 进入 loading；read-after-write 不一致恢复旧值并显示区域错误。
- 权限拒绝提供打开正确系统设置面板的操作；危险 quick action 说明不可逆影响。
- 搜索结果跳转后高亮并聚焦目标 setting；无结果可清除查询。

### Terminology / Token Calculator / Hardware

- Terminology 长词表虚拟化，搜索关键字高亮但不破坏 screen reader 文本；复制状态按条目隔离。
- Token Calculator 输入错误就地说明；货币、日期、token 数使用 `Intl`；stale 定价显示更新时间。
- Hardware 对比列在窄宽度横向滚动或切换详情，不压缩到文字重叠；数据来源和更新时间可见。

## 12. U07：Updater、启动问题和全局对话框

修改入口：

- `src/features/updater/`、`src/components/common/UpdateDialog.tsx`
- `src/components/common/{SettingsDialog,AboutDialog,CloseBehaviorDialog,StartupIssuesAlert}.tsx`

Updater 状态必须完整：idle、checking、upToDate、available、downloading、cancelling、installing、readyToRestart、error、unsupported。

要求：

- 自动检查安静执行；只有 available 或需要用户处理的 error 才主动提示。离线自动检查不弹阻断对话框。
- 手动检查立即打开对话框并显示 checking；重复点击由程序化 guard 合并。
- Release notes 有最大高度和独立滚动，纯文本/安全 Markdown；链接只允许 HTTPS 并用系统浏览器打开。
- 下载真实 total 可用时显示百分比和字节；未知 total 不显示假百分比。取消进入 cancelling，等待后端确认。
- installing 阶段说明应用即将重启/暂不可取消；readyToRestart 提供“立即重启”和“稍后”。
- 错误按 network、manifest、signature、disk、permission、platform 分类，给出对应恢复动作；不能向用户直接显示 Rust debug。
- StartupIssuesAlert 可展开受影响模块，支持 retry/restart/打开数据目录；敏感路径和密文不出现。
- About/Settings/Close dialogs 在中文、英文和 960×600 下不溢出，焦点 trap、Esc 和关闭后焦点恢复正确。

## 13. U08：响应式、国际化、可访问性与隐私

### 响应式

必测 viewport：960×600、1280×800、1440×900、1920×1080。Windows 额外测试 125%/150% scaling；macOS 测试 Retina 和系统增大文本。

- 页面根 `h-full min-h-0 overflow-hidden`；只有内容区滚动。
- 主内容 `min-w-0 flex-1`，工具栏 `shrink-0`；固定 board/toolbar/button 使用稳定尺寸。
- 关键文本不能被永久 truncate；名称可 truncate + Tooltip，错误后果和确认说明必须换行可读。
- 不按 viewport 缩放字体，不使用负 letter-spacing，不让最长英文单词越界。
- Modal/Sheet 最大高度不超过 viewport，header/footer 固定，body 独立滚动。

### 国际化

- 所有可见文案来自 zh/en locale；新增 key 同步，禁止顶层 `t()`。
- 日期、相对时间、数字、文件大小、货币使用 `Intl`，不手拼中文单位。
- 用测试伪翻译把英文扩展 35%，并用 CJK/emoji/超长 URL fixture 检查溢出；伪 locale 不必作为用户设置发布。
- canonical value 与展示文案分离；平台/provider/status 不用中文原值做逻辑判断。
- 切换语言后当前页面、Dialog、菜单、空态和错误即时更新，不要求重启。

### 可访问性

目标为 WCAG 2.2 AA 的适用部分：

- 正文/控件文字对比至少 4.5:1，大文字和非文本 UI 至少 3:1；明暗主题都检查。
- 所有操作可用键盘；focus 顺序与视觉顺序一致，focus ring 不被 `overflow-hidden` 裁掉。
- icon-only button 有 `aria-label` 和 Tooltip；自定义 Switch/Checkbox/Tab/Listbox 使用正确 role/state。
- loading、partial、error、copy success 使用克制的 `aria-live`；高频进度不每个字节播报。
- Dialog/Sheet 打开聚焦标题或首个安全控件，关闭恢复触发点；危险确认默认焦点不放在 destructive action。
- 图表/颜色状态提供文字等价；reduced motion 下停止 pulse 位移、scramble 和非必要动画。
- 引入 `axe` 自动化检查共享组件和核心页面，但自动化不能替代键盘/屏幕阅读器 smoke。

### 隐私与敏感数据

- password/token/cookie 不进入 DOM attribute、日志、toast、analytics 或截图 fixture。
- 密码 reveal 有明确范围和 TTL；复制后条件清剪贴板，不能覆盖用户后复制的内容。
- OAuth URL 只展示 origin/应用名，移除 code/token/state/query。
- 用户路径默认 Home 相对化；诊断/导出前显示脱敏范围。
- 不在 2.0.0 临时加入遥测。需要产品指标时先写隐私、开关、留存和删除设计。

## 14. U09：性能与视觉回归

### 性能预算

在固定 reference fixture 和机器信息下记录结果；不同 runner 不直接比较绝对时间。

| 场景 | 2.0.0 预算 |
|------|------------|
| 路由/Tab 点击到稳定反馈 | 100 ms 内出现选中或 loading 反馈 |
| 搜索键入到列表更新 | 50 ms 目标，P95 不超过 100 ms |
| 普通点击主线程长任务 | 不超过 100 ms |
| 500/2000 项列表 DOM | 仅渲染 viewport + overscan，不随总量线性增长 |
| 首次数据加载 | 立即 skeleton；网络/扫描时间不阻塞壳层 |
| 刷新 | 保留旧数据和滚动；无整页白闪 |
| 页面切换布局偏移 | 工具栏、主列表、详情无可见跳动 |

使用 React Profiler/浏览器 Performance/Playwright trace 找出重复 render、同步大计算和未虚拟化列表。不要为了通过预算机械添加 `useMemo`；先确认昂贵计算或引用导致真实重渲染。

### 视觉回归

必须建立 Playwright 或等价的浏览器态视觉项目，使用 mock repository 渲染纯前端状态；Tauri/WebView/OS 对话框另做目标平台 smoke。若不用 Playwright，替代方案必须同时提供多 viewport、截图 diff、键盘操作、DOM/overflow 和像素非空断言。

每个核心页面至少截图：

- 1280×800 light zh ready
- 1280×800 dark en ready
- 960×600 zh long text
- initialLoading、empty、filterEmpty、partial、failed、unsupported
- Dialog/Sheet 打开、危险确认、键盘 focus

截图前固定时区、字体、动画、随机数据和时间。测试除 snapshot diff 外还应断言：无水平溢出、关键元素 bounding box 不相交、Canvas/图片非空、文本位于容器内。
