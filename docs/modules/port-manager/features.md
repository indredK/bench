# Port Manager 功能说明

> 视角：产品 / 用户能力。技术实现见 [design.md](./design.md)；迭代规划见 [roadmap.md](./roadmap.md)。

## 1. 一句话定义

**一个端口与进程管理器——查看本机或远程主机的端口占用情况，定位占用进程，并安全释放端口。**

bench 不仅列出端口表，还把端口、进程、命令行串成完整的"占用链路"，并支持虚拟化大列表、端口历史回放、占用告警等开发者常用能力。

## 2. 解决的问题

| 痛点 | bench 的解法 |
|------|-------------|
| `lsof -i :8080` 命令繁琐，记不住参数 | 输入端口号一键查询 |
| 端口被占用但不知道是哪个进程 | 自动展开进程树，显示完整命令行 |
| 远程开发时本机看不到远程端口 | 远程检测模式（输入主机 + 端口） |
| 端口突然被占用要主动发现 | 占用告警：30 秒轮询 + 通知中心提醒 |
| 经常切换几个常用端口 | 端口历史：最近 10 个端口快速复用 |
| 大量 PID 列表渲染卡顿 | 虚拟化列表，仅渲染可视区域 |
| kill 进程怕误杀 | 二次确认对话框 + 后果说明 |

## 3. 已实现功能

### 3.1 端口扫描 — Local 模式

**入口**：开发工具箱 → 端口管理 → 顶部输入框输入端口号 → 回车。

**功能**：
- 输入端口号（如 `8080`），查询本机该端口占用情况
- 自动展示：
  - 端口状态（已占用 / 空闲）
  - 占用进程名 + PID + 完整命令行
  - 进程父子关系（树形展开）
  - 进程用户、启动时间、内存占用
- 支持通配符 / 范围查询（如 `8000-8100`）

### 3.2 端口扫描 — Remote 模式

**入口**：顶部切换按钮「本地 / 远程」→ 选「远程」→ 输入远程主机地址 → 输入端口号。

**功能**：
- 通过 SSH / TCP 探测远程主机端口占用情况
- 远程模式下隐藏 kill 按钮（不可安全释放远程进程）
- 支持主机保存与历史记录

**适用场景**：
- 远程开发时排查服务器端口占用
- CI 环境端口冲突远程诊断
- 多人协作时确认他人是否占用端口

### 3.3 进程 Kill（仅 Local 模式）

**入口**：端口扫描结果行 → "Kill" 按钮。

**功能**：
- 点击 Kill 触发 `DestructiveConfirmDialog` 二次确认
- 确认对话框展示后果说明：「即将终止进程 `<name>` (PID xxx)，可能导致正在运行的服务中断」
- 确认后调用系统命令终止进程
- 终止后自动刷新端口扫描结果
- Kill 操作文案全量 i18n（中英双语）

**安全设计**：
- 远程模式隐藏 kill 按钮
- 二次确认防误触
- 失败时 toast 显示具体错误（如权限不足）

### 3.4 端口历史

**入口**：输入框右侧"历史"按钮 → 弹出最近 10 个端口列表。

**功能**：
- 每次扫描成功后自动加入历史
- 历史按时间倒序排列（最新在前）
- 仅保留最近 10 个端口（超出自动剔除最早）
- 点击历史项快速填入输入框并触发扫描
- 历史持久化在 localStorage（`bench.port-manager.history`），重启 bench 不丢失

**适用场景**：
- 经常切换几个固定端口（如 dev server、API server、proxy）
- 重启后快速恢复上次工作上下文

### 3.5 虚拟化列表

**功能**：
- 使用 `@tanstack/react-virtual` 的 `useVirtualizer` 实现动态行高虚拟化
- 仅渲染可视区域的行 + 上下缓冲区
- 滚动时自动测量行高并调整
- 支持 `scrollToIndex`：从端口历史点击时自动滚动到对应行

**适用场景**：
- 大量 PID 列表（如 1000+ 进程）渲染不卡顿
- 端口范围查询（`8000-8100`）返回大量结果时仍流畅

### 3.6 占用告警

**入口**：端口列表顶部 Bell / BellOff 图标。

**功能**：
- 点击 Bell 启用告警，bell 图标变填充
- 30 秒轮询当前端口占用状态
- 与基线对比：
  - 端口从空闲变为占用 → 推送系统通知「端口 8080 已被进程 xxx 占用」
  - 端口从占用变为空闲 → 推送系统通知「端口 8080 已释放」
- 点击 BellOff 禁用告警，停止轮询
- 通知通过 `@tauri-apps/plugin-notification` 发送，进入系统通知中心

**适用场景**：
- 等待某个端口释放后自动开始下一步操作
- 监控关键端口（如 dev server）是否意外崩溃
- 排查"端口突然被占用"的偶发问题

### 3.7 远程模式 UI 行为

切换到 Remote 模式时：
- 显示远程主机输入框
- 隐藏 kill 按钮（远程进程不可安全释放）
- 历史记录与本地共用
- 占用告警仍可用（基于远程检测结果）

## 4. 用户界面要点

### 4.1 顶部控制栏

```
[本地 ▼ | 远程]  [端口号输入框]  [🔍 扫描]  [📜 历史]  [🔔 告警]
```

### 4.2 端口列表

```
┌──────────────────────────────────────────────────────────────┐
│ 端口 │ 状态 │ 进程名 │ PID │ 命令行 │ 用户 │ 启动时间 │ 操作  │
├──────────────────────────────────────────────────────────────┤
│ 8080 │ 占用 │ node  │ 1234│ node server.js │ user │ 10:00 │ Kill │
│ 8080 │ 占用 │ node  │ 1235│ node worker.js │ user │ 10:01 │ Kill │
└──────────────────────────────────────────────────────────────┘
```

### 4.3 Kill 确认对话框

```
┌─────────────────────────────────────┐
│ ⚠️ 确认终止进程                      │
├─────────────────────────────────────┤
│ 即将终止进程 node (PID 1234)         │
│ 这可能导致正在运行的服务中断          │
│                                     │
│ [取消]  [终止进程]                   │
└─────────────────────────────────────┘
```

## 5. 安全设计

| 层面 | 措施 |
|------|------|
| Kill 操作 | `DestructiveConfirmDialog` 二次确认 + 后果说明 |
| 远程模式 | 隐藏 kill 按钮，避免误杀远程进程 |
| 网络探测 | 远程检测仅 TCP connect，不发送任何凭证 |
| 通知 | 通知文案不包含敏感信息（仅端口 + 进程名） |
| 历史记录 | 仅存端口号，不存主机凭证 |
